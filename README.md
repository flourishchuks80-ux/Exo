## The Problem

> **"In February 2025, OpenAI pushed a backend update that wiped years of user memory data without warning."**
> *Affected users lost months of conversation history. Many accounts were never fully restored. No explanation. No recovery path. No accountability.*

Every AI you use today has memory. **None of it belongs to you.**

When you use ChatGPT, OpenAI stores what it learns about you on their servers. When you switch from ChatGPT to Claude, your AI starts from zero. When you switch from Claude to Gemini вҖ” same thing. Your context, preferences, project history, communication style, and domain knowledge are trapped inside whichever platform you used last.

You are renting an AI. You own nothing.

Every major AI memory system вҖ” Mem0, Zep, LangMem, Letta вҖ” stores data in centralized infrastructure. The platform controls it entirely. Users have no versioning, no automatic backup, and no real export capability. The AI memory market has already reached **$6.27 billion in 2026** and is projected to hit **$28.45 billion by 2030**. Every dollar in that market flows through platforms that can delete your data on a Tuesday afternoon.

Balaji Srinivasan identified this in February 2026:

> *"The fundamental question is whether AI stays on the leash. Private keys вҖ” not prompts вҖ” should determine who controls AI systems. Not your keys, not your bots."*

## Architecture вҖ” 6 Arkiv Entity Types

Every entity carries `PROJECT_ATTRIBUTE: { key: "app", value: "exo:v1" }` вҖ” applied on every `createEntity` call and every `buildQuery` call, without exception.

### SemanticMemory вҖ” *Permanent facts about you*

```
Attributes: app, type="semantic", topic, importance (1вҖ“100), agentId,
            confirmed (0|1), createdAt, updatedAt
Payload:    { content, source, confidence, tags, relatedKeys }
TTL:        Never вҖ” your identity doesn't expire
```

15-topic taxonomy: `profession` В· `expertise` В· `project` В· `preference` В· `communication_style` В· `goal` В· `constraint` В· `relationship` В· `location` В· `schedule` В· `health` В· `finance` В· `learning` В· `values` В· `background`

### EpisodicMemory вҖ” *Records of past conversations*

```
Attributes: app, type="episodic", agentId, sessionId, importance,
            topic, createdAt, sessionDate (YYYYMMDD for range queries)
Payload:    { summary, keyDecisions, openThreads, emotionalContext,
              linkedSemanticKeys }
TTL:        90 days вҖ” episodes fade; the important parts become semantic
```

### Instruction вҖ” *Your standing orders to any AI*

```
Attributes: app, type="instruction", scope, agentId, priority (1вҖ“10),
            isActive (0|1), category, createdAt
Payload:    { instruction, rationale, examples, negativeExamples }
TTL:        Never вҖ” instructions persist until you delete them
```

Categories: `tone` В· `format` В· `behavior` В· `constraint` В· `trigger`

### ContextDocument вҖ” *Your personal knowledge base*

```
Attributes: app, type="document", docType, title, tags,
            sizeBytes, createdAt, updatedAt
Payload:    { title, content, summary, sourceUrl, wordCount, language }
TTL:        365 days вҖ” documents expire annually; user renews
```

### AccessGrant вҖ” *Time-scoped memory sharing*

```
Attributes: app, type="grant", granteeWallet, scope, tokenHash,
            grantedAt, purpose
Payload:    { encryptedDek, allowedTopics, grantNote, accessLog }
TTL:        Configurable 1вҖ“90 days вҖ” enforced by Arkiv
```

### MemorySnapshot вҖ” *Versioned backup*

```
Attributes: app, type="snapshot", version, semanticCount, episodicCount,
            totalEntities, createdAt
Payload:    { aiSummary, topTopics, recentThemes, memoryHealthScore,
              entityKeyIndex }
TTL:        365 days вҖ” one year of snapshot history
```

## Quick Start вҖ” 5 Minutes

### Prerequisites

- Node.js 18+
- A [Privy](https://dashboard.privy.io) App ID (free вҖ” create an app, copy the App ID)
- An [Anthropic API key](https://console.anthropic.com) for Claude (the primary AI model)

### Setup

```bash
git clone https://github.com/flourishchuks80-ux/Exo.git
cd Exo
npm install
cp .env.example .env.local
```

Open `.env.local` and fill in the two required values:

```bash
# Required вҖ” get at https://dashboard.privy.io
NEXT_PUBLIC_PRIVY_APP_ID=your_privy_app_id

# Required вҖ” get at https://console.anthropic.com
ANTHROPIC_API_KEY=sk-ant-...

# Optional вҖ” needed only for the model switcher (GPT-4o and Gemini)
OPENAI_API_KEY=sk-...
GOOGLE_GENERATIVE_AI_API_KEY=...

# Set to your deployment URL (or leave as-is for local dev)
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

```bash
npm run dev
# вҶ’ http://localhost:3000
```

That's it. Connect your wallet via the landing page, and Exo will derive your master key from a wallet signature. You're ready to write your first memory to Arkiv Braga.

import { createPublicClient, http } from '@arkiv-network/sdk';

const client = createPublicClient({

## Arkiv Query Patterns Used

Exo demonstrates all major Arkiv query capabilities:

```typescript
// Equality filter + numeric range (importance вүҘ 60)
.where([eq('app','exo:v1'), eq('type','semantic'), eq('confirmed',1)])
.where([gte('importance', 60)])
.orderBy('importance', 'number', 'desc')

// Date range query (episodes in last 30 days)
.where([eq('app','exo:v1'), eq('type','episodic')])
.where([gte('createdAt', thirtyDaysAgo)])
.orderBy('createdAt', 'number', 'desc')

// Priority range (instructions вүҘ priority 8)
.where([eq('app','exo:v1'), eq('type','instruction'), eq('isActive',1)])
.where([gte('priority', 8)])
.orderBy('priority', 'number', 'desc')

// Cross-wallet read (team memory with share grant)
.where([eq('app','exo:v1'), eq('type','semantic')])
.ownedBy(teammateWallet)

// Token hash lookup (no ownedBy вҖ” public grant verification)
.where([eq('app','exo:v1'), eq('type','grant'), eq('tokenHash', hash)])
```
